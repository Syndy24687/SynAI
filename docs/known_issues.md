# SynAI Known Issues

This document records important implementation issues encountered during development, their causes, and the adopted solutions.

---

## Issue 001 — Broken Python Virtual Environment After Project Rename

### Date
2026-07-10

### Symptom

Running:

```bash
pip install -r requirements.txt
```

produced:

```
Fatal error in launcher...
```

The launcher still referenced:

```
WillJarvis\.venv
```

instead of

```
SynAI\.venv
```

### Cause

The project folder was renamed after the virtual environment had already been created.

Python virtual environments store absolute paths to the interpreter.

### Resolution

Delete the `.venv` directory.

Create a new virtual environment:

```bash
python -m venv .venv
```

Activate it and reinstall dependencies.

### Status

Resolved

## Issue 002 — Incorrect Vosk Integration Generated by AI

### Date

2026-07-10

### Symptom

Speech recognition failed with:

```
TypeError:
initializer for ctype 'char *'
must be a cdata pointer
```

### Cause

The generated implementation passed microphone buffers incorrectly to `KaldiRecognizer.AcceptWaveform()`.

### Resolution

Rewrite the recognizer using the official Vosk streaming example with `queue.Queue` and a `RawInputStream` callback.

### Status

Resolved

## Issue 003 — Missing Speech Model

### Date

2026-07-10

### Symptom

```
Model path not found
```

### Cause

The Vosk model had not yet been downloaded and the `.env` file did not specify the model path.

### Resolution

Download `vosk-model-small-en-us-0.15`, place it under `models/`, and configure:

```
SPEECH_MODEL_PATH=./models/vosk-model-small-en-us-0.15
```

### Status

Resolved
